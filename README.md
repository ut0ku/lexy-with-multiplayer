# Lexy with multiplayer

Веб-приложения для изучения иностранных слов методом интервального повторения с мультиплеерным взаимодействием.

## Возможности
Основное веб-приложение:
- Регистрация и авторизация пользователей (JWT)
- Просмотр публичной библиотеки колод
- Добавление публичных колод к себе в библиотеку
- Создание, редактирование, удаление своих колод
- Добавление, редактирование, удаление карточек в колодах (слово / перевод)
- Подача заявки на публикацию колоды в публичную библиотеку
- Режим изучения «Знаю / Не знаю»
- Режим изучения с вводом ответа с клавиатуры
- Добавление карточки в избранное
- Отдельная колода / раздел с избранными карточками
- Вкладка «Статистика»: серия занятий, календарь активности, общее время, точность ответов
- Профиль: управление данными, тема оформления, включение/отключение push-уведомлений
- Панель администратора: назначение ролей, блокировка пользователей (причина, срок)
- Панель администратора: создание колод напрямую в библиотеке
- Панель администратора: проверка и публикация колод по заявкам

Микросервис мультиплеера:
- Вкладка «Мультиплеер» в основном приложении
- Создание игровой сессии
- Приглашение пользователей в сессию
- Выбор колоды для игры
- Выбор режима: «Совместный» или «Соревновательный»
- Выбор варианта ввода ответа: «С клавиатуры» или «Знаю / Не знаю»
- Запуск игровой сессии создателем
- Ответы на карточки в реальном времени с синхронизацией
- Отображение прогресса и счёта участников в реальном времени
- Экран результатов после прохождения колоды
- Сравнение своего результата с результатами других участников сессии

## Использованные технологии:
- **Frontend:** React 19 со сборщиком vite
- **Backend:** Node.js с фреймворком Express.js
- **Real-time:** WebSocket Socket.IO
- **База данных:** PostgreSQL "lexy" и "multiplayer"
- **Контейнеризация:** Docker (Docker Compose)
- **Аутентификация:** JWT-токен

## Быстрый запуск:
1. Установить Docker Desktop
2. Выполнить запуск контейнеров (предварительно перейдя в папку проекта), командой:

```bash
docker compose up -d
```

3. Дождаться запуска всех процессов:

Фронтенд `http://localhost:80`.

Основной сервер `http://localhost:3002`.

Микросервис мультиплеера `http://localhost:3003`.

PostgreSQL `http://localhost:5432`.
